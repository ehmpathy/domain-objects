wish =

existed a type called `HasReadonly`

which looks at the `public static metadata` and `public static readonly` attributes declared on the static properties of DomainObjects

and marks them all as required

---

it should use the default metadata if metadata is not expxlicitly specified

there should be no default readonly if not explicitly specified

---

read the details of DomainEntity.readonly and DomainObject.metadata for knowledge on metadata vs readonly

---

use the DeclaredAwsRdsCluster example to prove HasReadonly is effective

e.g.,

```ts
interface DeclaredAwsRdsCluster {
  arn?: string;
  name: string;
  port?: number
}
export class DeclaredAwsRdsCluster extends DomainEntity<DeclaredAwsRdsCluster> implements DeclaredAwsRdsCluster {
  public static primary = ['arn']
  public static unique = ['name']
  public static metadata = ['arn']
  public static readonly = ['port']
}
const cluster: HasReadonly<DeclaredAwsRdsCluster> = {
  arn: '1234',
  name: 'testdb',
  port: 821,
}
const port: number = cluster.port;
```
