# blocker.1: Mutable state via assignment and push

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/manipulation/omitReadonly.ts:20

The splitReadonlyPaths function builds direct and nested using const arrays/objects then mutates them via .push() and direct property assignment inside forEach. This violates the immutability rule which requires const + spread/clone/reduce instead of assignment or mutation. Mutable intermediate state is a maintenance hazard that makes code harder to reason about, test, and parallelize.

**snippet**:
```ts
  const direct: string[] = [];
  const nested: Record<string, string[]> = {};
  keys.forEach((key) => {
    const dotIndex = key.indexOf('.');

    // no dot => direct readonly key on this object
    if (dotIndex === -1) {
      direct.push(key);
      return;
    }

    // dot => nested path; group by head segment, keep the rest (tail) of the path
    const head = key.slice(0, dotIndex);
    const tail = key.slice(dotIndex + 1);
    nested[head] = [...(nested[head] ?? []), tail];
  });
  return { direct, nested };
```

# blocker.2: Generic Error hides context (failloud violation)

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/manipulation/hasReadonly.ts:35

hasReadonly throws a plain new Error for non-DomainObject input instead of UnexpectedCodePathError (or similar) with metadata. This violates failloud (errors must include actionable context: what failed, why, how to fix, relevant values) and failfast guidance to use helpful-errors subclasses. Plain errors lead to silent defects or extra debug hours.

```ts
    // ensure it's a domain object

```
