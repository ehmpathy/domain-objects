# nitpick.1: Fake verification in type tests

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.test/pitofsuccess.errors/rule.forbid.failhide.md

**locations**:
- src/manipulation/HasReadonly.type.test.ts

Tests for type errors use @ts-expect-error + assignment + expect(x).toBeDefined(). This is a runtime silent pass-through with no actual verification (always passes), similar to forbidden fake patterns like if (!cond) { expect(true).toBe(true) }. The runtime path has no verification of the expected condition.

**snippet**:
```ts
      it('should type-error when required readonly keys are missing', () => {
        // @ts-expect-error: Property 'port' is missing
        const cluster: HasReadonly<typeof DeclaredAwsRdsCluster> = {
          arn: '1234',
          name: 'testdb',
        };
        expect(cluster).toBeDefined();
      });
```
